Commit (Git)

A commit is a snapshot of a Git repository. Commits should be short, representing an atomic change.

Commit message

A commit message can have a short summary, and an optional longer explanation after an empty line. Do not use emoji's in commit messages like [gitmoji](https://gitmoji.dev/) advocates. Emojis do not provide any benefit to the commit message. There are still a lot of tools that lack proper emoji support. It is also way more difficult to filter or search commit messages when they use emojis for describing the commit.

Use the following prefixes for commit message titles:

  • feat: for new functionality.
  • fix: for reworking existing code to solve bugs.
  • refact: for refactoring code.
  • docs: for adding documentation.
  • test: for adding or fixing tests.
  • bump: for incrementing version numbers of dependencies.

A good convention to use is conventional commits. The commitlint project expands on this and adds more types.

It is important to not introduce to many prefix categories. This will defeat the purpose if categorizing commits.
